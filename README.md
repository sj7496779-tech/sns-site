# SNS Site

## プロジェクト概要
Djangoで構築された、社内向けのWebアプリケーションです。

本アプリには主に2つの機能があります。

- 社内チャット機能: 投稿、返信、リアクション、検索などのチャット掲示板機能
- ブックメーカー機能: お題作成、ポイントの賭け、オッズ計算、結果確定、配当

## 機能

### 掲示板
- チャットの閲覧、投稿、返信、削除が可能
- 投稿にはテキストに加え、画像添付が可能
- 投稿へのリアクション（いいね）を ON/OFF できる

### ブックメーカー
- お題の作成と複数の選択肢設定が可能
- 各お題に対して、締め切り日時を設定できる
- 受付中のお題に対してポイントを賭けられる
- 賭け金に応じたオッズを自動計算し、各選択肢の倍率を表示する

### 結果確定と配当
- お題作成者は締め切り後に正解選択肢を確定できる
- 強制終了機能により、期限前でも受付を終了して結果確定へ移行可能
- 正解選択肢に賭けたユーザーには総プールに応じたポイント配当を行う
- 不正解の賭けには配当されない

### ランキング
- `AccountProfile.currentpoint` に基づく上位10ユーザーランキングを表示する
- 自分の保有ポイント順位を確認しながら賭けを楽しめる

## 使用技術スタック

- Python 3
- Django
- SQLite
- HTML / CSS / JavaScript

## ディレクトリ構成

- `manage.py` - Django 管理コマンド
- `sns_project/` - プロジェクト設定
  - `settings.py` - アプリ設定と静的・メディア設定
  - `urls.py` - グローバル URL ルーティング
- `bookmaker/` - アプリ本体
  - `models.py` - データモデル定義
  - `views.py` - ビューと画面制御
  - `services.py` - ビジネスロジック
  - `urls.py` - アプリ固有 URL
  - `templates/bookmaker/` - ブックメーカー画面テンプレート
- `templates/` - 共通テンプレート（ログイン等）
- `static/` - CSS など静的リソース
- `media/` - 投稿画像の保存先

## モデル説明

- `AccountProfile`
  - `User` に紐づくプロフィールモデル
  - ユーザーの保有ポイントを `currentpoint` で管理する
- `Topic`
  - ブックメーカーのお題を表すモデル
  - タイトル、詳細、期限 `deadtime`、ステータス `status` を持つ
  - `is_active` / `is_closed` プロパティで受付中・終了を判定する
- `Option`
  - `Topic` の選択肢を表すモデル
  - `text` と `is_correct` を持ち、`odds` で賭け比率を算出する
- `Bet`
  - ユーザーが選択肢に賭けたポイントを記録するモデル
  - 同一ユーザーと同一選択肢の組み合わせは一意になる
- `Chat`
  - 掲示板投稿を表すモデル
  - 画像添付 `image` と、お題引用 `shared_topic` をサポートする
- `Reaction`
  - 投稿へのいいね / リアクションを保存するモデル
  - 同じユーザーが同じ投稿に複数回リアクションできない制約を持つ
- `Reply`
  - 投稿への返信を表すモデル
  - `parent` フィールドでネストした返信を管理する

## アーキテクチャ

- Django の MVC に近い構造で実装されている
  - `models.py`：データモデルと DB テーブル定義
  - `views.py`：リクエスト受信、処理、レスポンス生成
  - `templates/`：HTML 表示部分
  - `urls.py`：URL ルーティング管理
- `bookmaker/services.py` には、ビジネスロジックと再利用可能な処理が集約されている
  - ユーザーのプロフィールを取得・作成する
  - 新規お題の作成処理をまとめる
  - 受付中のお題・ランキング・ユーザーのベット履歴などの共通クエリを提供する
  - ビュー側の重複コードを減らし、処理を分離する
- 主要なデータフロー
  1. ユーザーがブラウザで URL にアクセスする
  2. `urls.py` で対応するビューが選択される
  3. ビューが `models` や `services` を呼び出し、DB からデータを取得・更新する
  4. テンプレートにデータを渡して HTML を生成し、ブラウザに返す
- 認証と権限
  - Django の標準ログイン・ログアウト機能を使用
  - 一部の操作（投稿作成、ベット、返信、リアクション）はログイン必須
- ストレージ構成
  - SQLite をデータベースとして使用
  - 画像ファイルは `media/` に保存
  - CSS など静的ファイルは `static/` で管理

## 使い方

1. リポジトリをクローンし、仮想環境を作成して有効化します。

```bash
python -m venv myenv
myenv\Scripts\activate
```

2. 依存関係をインストールします。

```bash
pip install -r requirements.txt
```

この `requirements.txt` には、本アプリを動作させるために必要な Python パッケージとバージョンが記録されています。

3. マイグレーションを作成・適用します。

```bash
django-admin makemigrations
python manage.py migrate
```

4. 管理ユーザーを作成します。

```bash
python manage.py createsuperuser
```

5. 開発サーバーを起動します。

```bash
python manage.py runserver
```

6. ブラウザで `http://127.0.0.1:8000/` にアクセスします。

7. ログイン後、以下の操作が可能です。

- トップ掲示板画面で投稿を閲覧、投稿、返信、リアクション
- 画像を添付して投稿
- 「ブックメーカー」画面でお題を作成
- 各選択肢にポイントを賭ける
- お題の期限切れ後、作成者が正解を確定すると配当が実行される
- 保有ポイントランキングを確認する

開発中にコードを修正した場合は、`manage.py runserver` を再起動するか、`python manage.py migrate` を再実行して変更を反映する必要があります。
